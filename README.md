# 基于 Agent 的服务自动化修复系统
## 挑战任务：
- 要求构建一个 Agent，它能够监控一个简单的 Web 服务。当服务报错或者有新的bug提交时，Agent 自动获取 Traceback 日志，通过大模型分析原因，自动修改代码并提交 PR，最后通过飞书卡片通知开发者：“我发现了一个 Bug 并已为您修复，请 Review”。
- Agent 需要具备环境感知的 Tool Use 能力（Read Log, Read Code, Run Test, Git Commit），
## 交付物：
1. 代码仓库：包含 Agent 的逻辑代码和其自动生成的修复记录。
2. 演示视频：展示从报错到 Agent 自动发出飞书通知并完成修复的全过程。
## 评判标准：
- 工具链整合能力：是否能熟练使用AI工具和AI框架，实现自动化研发运维和问题修复agent。
- 安全性考量：充分考虑“如何防止 AI 乱改代码导致更大的崩溃”。
